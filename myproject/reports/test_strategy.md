# Test Strategy

## Approach
Risk-based testing focusing on:
- Authentication
- Data consistency
- API reliability
- Failure handling

## Scope
- UI testing
- API testing
- GraphQL
- Performance
- Security



## Key Risks
1. Data inconsistency across services
2. Unauthorized access
3. Silent API failures

## Assumptions
- Systems are not integrated
- Data flow simulated

## Tools Used
- pytest
- requests
- pandas
- curl
